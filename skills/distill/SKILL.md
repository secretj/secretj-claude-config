# Distill — 세션 도메인 지식 축적

현재 세션 대화를 돌아보며 **비자명한 mailplug 도메인 지식**을 추출해 Obsidian에 저장한다.

## 저장 기준 (이것만 저장)

저장 O:
- 코드만 봐서는 알 수 없는 **정책·제약** (예: "토큰 만료 30분은 설정값 아니고 하드코딩")
- **크로스 저장소 영향** (예: "gw-member config 변경 시 wm70-api 재시작 필요")
- **비자명한 버그 원인·재현 조건** (예: "특정 상품코드에서만 500 발생")
- **배포 순서·동시 배포 제약** (예: "wm70 먼저 배포 후 gw-member")
- **환경·인프라 제약** (예: "인증존에서 외부 API 직접 호출 불가")
- **기획 정책 예외** (예: "GW_EDU 상품만 첨부파일 용량 2배")

저장 X:
- 자명한 코드 동작, 일반적인 PHP/CI4 동작
- 이번 세션에서 구현한 코드 내용 (코드 자체가 기록)
- 미확인·추정 사항

## 포맷 규칙 — 유형별 구조화

긴 산문 금지. **실제 동작하는 방식**을 조건·규칙·표로 기술한다.

### 유형 A — 노출/동작 정책 (조건이 있는 경우)

```markdown
### {기능명} 정책 (날짜[, 티켓])

**동작 조건** (하나라도 충족 시):
- 조건 1
- 조건 2

**비동작 조건** (모두 충족 시):
- 조건 1
- 조건 2

**예외**:
- 상황 → 결과

**최종 판정식**: `A OR (B AND C)` ← 복잡한 경우만
**관련 코드**: `ClassName::method()` — 한 줄 역할 설명
```

### 유형 B — 파일/경로 구조

```markdown
### {파일명} (날짜[, 티켓])

- 경로: `{home}/{domain}/{userId}/_CONTROL/{파일명}`
- 역할: 한 줄
- 생성 시점: 언제
- 내용 형식: 무엇이 들어있나
```

### 유형 C — 버그·함정 패턴

```markdown
### {문제 제목} (날짜[, 티켓])

- **증상**: 무엇이 잘못됨
- **원인**: 왜 그런가
- **주의**: 다음에 이 코드 건드릴 때 조심할 것
- **올바른 패턴**: `코드 예시`
```

### 유형 D — 컨벤션·규칙

```markdown
### {규칙 제목} (날짜)

- 규칙 1
- 규칙 2
- ❌ 하면 안 되는 것
```

### 유형 E — 크로스 저장소 의존

```markdown
### {의존 관계 제목} (날짜[, 티켓])

- **흐름**: A → B → C
- **트리거**: 어떤 변경이 전파를 일으키나
- **주의**: 한쪽만 배포 시 발생하는 문제
```

## 실행 절차

### 1. 세션 스캔
세션 대화 전체를 돌아보며 저장 기준 항목을 추출하고 유형(A~E) 분류.

### 2. 항목 없음 처리
```
[distill] 이번 세션에서 축적할 도메인 지식이 없습니다.
```

### 3. 사용자 확인
구조화된 미리보기를 보여주고 확인 받는다:
```
[distill] 다음 항목을 Obsidian에 저장합니다:

1. mailplug/wm70-api/smtp.md [신규] — 유형 A (정책)
   ### 앱 비밀번호 메뉴 노출 정책
   동작 조건: SMTP_ALLOW 파일 존재 / POP3 또는 IMAP 활성화
   비동작 조건: hide 셋업 모두 Y / 관리자 도메인 Off / 셋업+설정 모두 Off
   예외: 사용자 본인 Off → 노출 유지

2. mailplug/_common/deploy.md [추가] — 유형 E
   ...

저장할까요? (전체 / 번호 선택 / 취소)
```

### 4. Obsidian 저장

각 항목별:
1. `obsidian_search_notes`로 대상 경로 파일 존재 확인
2. 있으면 → `obsidian_append_to_note`로 섹션 추가
3. 없으면 → `obsidian_write_note`로 신규 파일 생성

**파일 헤더** (신규 생성 시):
```markdown
# {도메인} — {저장소}

> mailplug/{repo}/{도메인}.md
```

이후 내용은 유형별 포맷대로 작성.

### 5. 완료 보고
```
[distill] 저장 완료:
- mailplug/wm70-api/smtp.md (신규, 유형 A)
- mailplug/_common/deploy.md (+1, 유형 E)
```
